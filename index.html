Курс Git

Коротка інформація про системи контролю версій та Git

Системи контролю версій (Version Control Systems, VCS) дозволяють відстежувати зміни у файлах, координувати роботу команд та зберігати історію проекту. Git — це розподілена система контролю версій, що надає потужні інструменти для роботи з кодом, зберігання змін та кооперації у команді.

Основні можливості та переваги Git

  • Розподілена структура — кожен користувач має повну копію репозиторію.
  • Висока швидкість роботи.
  • Легке створення та злиття гілок (branching & merging).
  • Можливість відстежувати історію змін та відкотитися до попередніх станів.
  • Підтримка роботи у команді через віддалені репозиторії.

10 базових команд Git з коментарями

git init — створити новий Git репозиторій
git clone <url> — скопіювати існуючий репозиторій
git status — показати стан репозиторію
git add <file> — додати файл до staging area
git commit -m "message" — зафіксувати зміни з коментарем
git push — відправити зміни на віддалений репозиторій
git pull — отримати та об'єднати зміни з віддаленого репозиторію
git branch — показати або створити гілки
git checkout <branch> — перейти на іншу гілку
git merge <branch> — об'єднати гілку з поточною
